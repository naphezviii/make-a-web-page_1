Five way of thinking
like a programmer:

  1. Procedural thinking: creating perfectly clear and unambiguous instructions (sequential thinking)
  2. Abstract thinking: finding generality (=> abstractions) to avoid unnecessary repetition
  3. Systematic debugging: following a structured process to find the mistakes (bugs)
  4. Technological empathy: being aware that computers are powerful tools and at the same time stupid (no room for ambiguity)
  5. Systems thinking: breaking a big problem down into smaller components

Lesson 1:
Key aspects of the world wide web

World Wide Web

World Wide Web is a collection of Hypertext M arkup Language (HTML) documents that are stored on large computers called servers. These documents have two parts: first the meta data such as title of the pages and other elements like Cascading Style Sheets (CSS) and then the actual content of the page.
To access a document (a webpage) you have to use a web browser on a computer that sends a HTTP request to a specific server which serves the document back to your terminal.

Hypertext Markup Language (HTML)

The HTML determines the content (text) and the way it looks (markup). It can reference other content (images and video) as well as links to other HTML pages. These are HTML attributes.

Elements

A HTML page consists of elements. They are the building blocks: content bracketed with so called tags. It is not necessary to know all the tags. A good resource can be found here.

Span and div

It is important to note that some elements are either inline or block. This means they either flow with the text or have a hight and width that can be assigned around the content (invisible boxes).

Lesson 2:
Webpages, documents, and structure

Developer Tools

Developer tools for instance in Chrome (menu top right) allow to inspect the structure and elements (indicated with triangles) of the a webpage: (1) to view the visible as well as the invisible elements; (2) to inspect the dimensions; and (3) their style (CSS).

Tree structure: Document object model (DOM)

HTML pages are structured like an outline (tree-like structure). HTML elements can include other elements. So the more indented the element is the deeper nested it is.
This in turn means the structure is a series of nested boxes. Visually the tree structure is boxes in boxes, in boxes.

Classes

These boxes have 'labels' so called class attributes. Like other attributes a class is defined with a value.

Code Writing Tools

There are three key tools that differ in speed of feedback, ease of use, level of power,and the number of programming languages they can handle.

Scratchpad
easy to use (HTML & CSS) with immediate feedback, great for beginners or quick sketches, no option to share
Codepen
fewer beginner features slightly delayed feedback, easy to share (account needed), powerful editor for HTML, CSS and Javascript
Sublime
more difficult to use, feedback on refresh, complicated to share, very powerful editor for all programming languages

The recommendation is to start with Scratchpad and to share with Codepen, later with Sublime

Lesson 3:
Style, selectors, and boxes

Cascading Style Sheets (CSS)

CSS identifies (matches) a class or a tag of an element, and then determines what to do with it, or what property values to change. This is akin to a find and replace tool. The order (place and sequence of the rules) and specificity are important.

Order

A subsequent definition overwrites the previous one:
  1. browser’s the default style (differs from browsers to browser)
  2. separate stylesheet file (recommended)
  3. stylesheet inside the header of a HTML document (only for small projects)
  4. inline style in an element (not recommended)

Specificity

Rules affect all of an elements’ child elements, not only the elements they directly match (= cascading). If several, overlapping rules define a child element, the more specific rule applies. See inheritance and note that text-related properties are inherited, box-related properties not.

Selectors

  1. ID selectors (most powerful in terms of specificity: #selector
  2. Class selectors (most useful and versatile): .class-name
  3. Tag selectors (most useful when changing properties unique to tan element): e.g. h2
  4. Attribute selectors (even more useful than classes because they have the same specificity value, but can be any attribute not just class, plus they can have a value you can select by): e.g. [data-modal="open"]
Selectors can be combined together. A webpage covering all the selectors and properties can be found here

Box Sizing

HTML elements are boxes and each box has 4 components: content, padding, border and margin. Either set sizes in terms of percentages rather than pixels or set the box-sizing attribute to border-box for every element.
NB: Different browsers display the same code differently.

Box Positioning

Divs—as block elements (as opposed to inline)—use by default the entire page width. The rule ‘display: flex;’ lets divs appear next to each other. See this article for more details.

Best practice:
Code, test, refine, and verify

  1. Look for natural boxes in the proposed design
  2. Look for repeat styles and semantic elements
  3. Write the HTML
  4. Apply styles from biggest to smallest
  5. Fix things (used the browser dev tools to quickly test before executing in HTML)
